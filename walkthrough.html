PlombiPro: A Step-by-Step Build Process

This document walks you through building the PlombiPro app from start to finish.

The process relies on two key documents you've created:

  1. The "Brain": plombipro_ai_cli_prompt.md (This tells your Gemini CLI how to use the other 5 files).
  2. The "Map": plombipro_part5_final_summary.md (This provides the 7-phase master checklist for what to build).

Your workflow will be to command your Gemini CLI to execute the tasks from the "Map" (File 5), using the rules from the "Brain" (File 0) and the code from Files 1, 2, 3, and 4.

The Core Workflow

As defined in your plombipro_ai_cli_prompt.md, your interaction with the Gemini CLI should follow this pattern.

You (Developer):

FILES: [File(s) to use, e.g., 1, 3, 4]
TASK: [Your specific task, e.g., "Generate the complete QuoteFormPage"]
CONTEXT: [Any relevant info, e.g., "Mobile first, with 6 sections"]
OUTPUT: [e.g., "Full lib/screens/quote_form_page.dart file"]

The CLI (AI Assistant):

  • "Okay, I will reference FILE 4 for the exact AI prompt, FILE 1 for the layout, and FILE 3 for the service connections to generate lib/screens/quote_form_page.dart."

Let's apply this workflow to the 7 phases from your File 5 Master Checklist.

Phase 1: Local Development Setup (Week 1-2)

Your goal here is to set up the project structure, database, and environment.

1. Setup .env File

  • Your Command: "Gemini, set up my .env file."
  • CLI Action: "Got it. Referencing FILE 5 (Environment Variables Setup), I will generate lib/.env with all the required keys like SUPABASE_URL, SUPABASE_ANON_KEY, and STRIPE_PUBLISHABLE_KEY."

2. Create Database Schema

  • Your Command: "Gemini, I need the SQL to create all 12 database tables."
  • CLI Action: "Understood. I'll analyze the models in FILE 3 (Models: Quote, Client, Product, etc.) and the Supabase functions in FILE 3 (SupabaseService) to generate the complete SQL schema, including table definitions, relationships (foreign keys), and RLS policies."

3. Configure main.dart

  • Your Command: "Gemini, set up my main.dart file for initialization."
  • CLI Action: "Referencing FILE 5 (EnvConfig), I'll generate the main.dart file, including dotenv.load(), Supabase.initialize(), and Stripe.publishableKey setup before runApp()."

Phase 2: Core Features Development (Week 3-6)

This is where you'll use the AI prompts from File 4 to build the UI.

1. Generate the HomePage

  • Your Command: "Gemini, generate the HomePage."
  • CLI Action: "Okay. As per your ai_cli_prompt.md rules, I will use the exact FlutterFlow AI prompt from FILE 4 (Prompt 1: HomePage/Dashboard). This prompt references the layout from FILE 1 and will be used to generate lib/screens/home_page.dart."

2. Generate the QuotesListPage

  • Your Command: "Gemini, generate the QuotesListPage."
  • CLI Action: "Following the same process, I'll use the exact prompt from FILE 4 (Prompt 2: QuotesListPage) to generate lib/screens/quotes_list_page.dart, which includes the search bar, filter chips, and QuoteCard layout from FILE 1."

3. Implement Calculation Logic

  • Your Command: "Gemini, I'm building the QuoteFormPage and need the calculation logic."
  • CLI Action: "Perfect. Referencing FILE 3 (Invoice Calculation Service), I'll provide the InvoiceCalculator class. You will call InvoiceCalculator.calculateTotals() from your form page to get the ht, tva, and ttc values."

4. Implement Supabase CRUD

  • Your Command: "Gemini, how do I fetch all quotes?"
  • CLI Action: "Referencing FILE 3 (Supabase Service), you will use the SupabaseService class. The function is fetchQuotes(), which already includes the logic to select from the quotes table and join quote_items and clients."

Phase 3: Advanced Features (Week 7-8)

Here you'll combine backend (File 2), frontend services (File 3), and UI (Files 1 & 4).

1. Deploy OCR Backend

  • Your Command: "Gemini, deploy the OCR Cloud Function."
  • CLI Action: "Okay. Referencing FILE 2 (Cloud Function 1: OCR Invoice Processing), I will provide:
  1. The complete Python code for ocr_process_invoice.
  2. The requirements.txt dependencies.
  3. The exact gcloud functions deploy... command from FILE 2 (Deployment Commands), including the correct memory, timeout, and env vars."

2. Build ScanInvoicePage UI

  • Your Command: "Gemini, generate the ScanInvoicePage."
  • CLI Action: "Using the exact prompt from FILE 4 (Prompt 4: ScanInvoicePage), I will generate lib/screens/scan_invoice_page.dart. This page is pre-configured to use the layout from FILE 1 and call the OcrService from FILE 3, which in turn calls the Cloud Function you just deployed."

3. Implement Stripe Payments

  • Your Command: "Gemini, guide me through setting up a payment."
  • CLI Action: "This is a 3-part process defined by your files:
  1. Backend: First, we deploy the create_payment_intent Cloud Function from FILE 2.
  2. Service: We then use the StripePaymentService from FILE 3 in our Flutter app.
  3. UI: The service is called from the InvoiceFormPage (defined in FILE 1), which triggers the Stripe.instance.presentPaymentSheet()."

Phase 4: Notifications & Scheduling (Week 9)

1. Deploy Email Function

  • Your Command: "Gemini, deploy the email notification function."
  • CLI Action: "Copying from FILE 2 (Cloud Function 4: Send Email Notification), here is the Python code for send_email_notification and the gcloud command to deploy it. This function uses the SendGrid API key from your environment variables."

2. Send a Quote via Email

  • Your Command: "Gemini, how do I send a quote from my app?"
  • CLI Action: "Referencing FILE 3 (Email Service), you will call the EmailService.sendQuoteEmail() method. This function calls the backend function you just deployed, passing the client's email, quote number, and the PDF bytes (which you get from the PdfGenerator service, also in FILE 3)."

Phases 5-7: QA & Deployment (Week 10)

Your AI CLI becomes a guide and a checklist-keeper, using File 4 and File 5.

1. Run QA Checks

  • Your Command: "Gemini, what do I need to test for Phase 5?"
  • CLI Action: "Referencing the FILE 5 (Master Checklist - Phase 5), your QA list includes:
  • Functional: 'Create quote → Convert to invoice → Send email ✓'
  • Performance: 'App launch time < 2 seconds'
  • Security: 'RLS policies prevent unauthorized access'
  • Responsive: 'Mobile (360px width) - all pages ✓'
    I can help you write tests for each of these."

2. Prepare for iOS Deployment

  • Your Command: "Gemini, walk me through the iOS deployment."
  • CLI Action: "Following the FILE 4 (Complete iOS Deployment Guide), the first steps are:
  1. Run xcode-select --install and flutter doctor.
  2. Run pod install --repo-update in the ios folder.
  3. Next, I will generate the exact Info.plist content from FILE 4 (Step 3: Configure Info.plist), which includes the necessary permissions for NSCameraUsageDescription (for OCR)."

3. Prepare for Android Deployment

  • Your Command: "Gemini, help me with the Android build."
  • CLI Action: "Following FILE 4 (Complete Android Deployment Guide), I'll provide the contents for android/app/build.gradle (setting compileSdkVersion 33) and AndroidManifest.xml (adding INTERNET and CAMERA permissions)."

This step-by-step process, combining your project plan (File 5) with your AI interaction rules (File 0), will allow you to build the app efficiently, feature by feature.
